# Security Policy

## Supported Versions

Security fixes are handled on the active development branch and the latest
public release.

## Reporting A Vulnerability

Please avoid opening a public issue with full exploit details.

Preferred flow:

1. Use GitHub private vulnerability reporting if it is enabled for the repository.
2. If private reporting is not available, open a minimal issue requesting a
   secure contact path without including sensitive reproduction details.

Please include:

- affected platform
- affected version or commit
- steps to reproduce
- impact
- any proof-of-concept details needed to confirm the issue

We will try to confirm the report, assess impact, and coordinate a fix before
public disclosure.
